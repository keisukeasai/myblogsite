# Editorial Guide — 記事執筆ガイド

記事を書く人が迷わないための手引き。フロントマター・構成・トーン・公開前チェックまで。

---

## 1. 記事ファイルの置き場所

`src/content/blog/` に `.md`（通常）または `.mdx`（式・コンポーネントを埋め込みたい場合）を置く。
**ファイル名がそのままURLのスラッグ**になる（例: `iphone-stand-review.md` → `/blog/iphone-stand-review/`）。英小文字・ハイフン推奨。

---

## 2. フロントマター（必須項目）

```yaml
---
title: 記事タイトル（32〜40字程度まで）
description: 一覧カード・検索・OGP・meta に使う説明。100字前後で内容が伝わるように。
pubDate: 2026-07-19
category: gadget   # 下記6つから1つ（必須・スラッグで指定）
tags: ["任意タグ", "自由に複数可"]
image: ./images/cover.png   # 任意。省略時はカテゴリ絵文字のプレースホルダ
imageAlt: 画像の内容を説明するテキスト（画像を入れる場合は必須級）
---
```

- `category` を間違える/書き忘れるとビルドエラーで気づける（安全装置）。
- `tags` は補助ラベル。カテゴリと重複しない、細かい切り口を自由に。

---

## 3. カテゴリの使い分け

| slug | 絵文字 | 名前 | 何を書くか |
|---|---|---|---|
| `gadget` | 📱 | ガジェット | ガジェット単体の紹介・使用感 |
| `interior` | 🪑 | インテリア | 家具・照明・部屋づくり |
| `desk` | 🖥️ | デスク環境 | デスク周り・在宅ワーク環境 |
| `storage` | 📦 | 収納 | 収納術・片付け・整理のアイデア |
| `review` | ⭐ | レビュー | 「買ってよかった」の正直レビュー |
| `life` | ☕ | 暮らし | 暮らし全般・エッセイ・お知らせ |

> 1記事1カテゴリ。迷ったら「読者がどの棚を探すか」で選ぶ。定義変更は [`src/data/categories.ts`](../src/data/categories.ts)。

---

## 4. 記事の構成（推奨）

1. **導入**（2〜4行）— 何の記事か・誰に向けてかを最初に示す。
2. **本文**（`##` = h2 で区切る）— 見出しは内容が分かる言葉に。必要なら `###`（h3）で細分化。
3. **まとめ**— 結論・どんな人に向くか。

- リスト・引用・コードブロックはそのまま使える（`prose` で自動整形）。
- 画像は本文中にも挿入可（`![alt](./images/xxx.png)` / MDXなら `<Image>`）。alt は必ず書く。

---

## 5. レビュー記事のトーン（正直レビューの型）

商品レビューでは、最低この3点に触れる:

- **良かったところ**
- **気になったところ**（短所を隠さない）
- **どんな人に向いているか**

「便利だから」ではなく **“毎日が少し心地よくなるか”** を判断軸にする（→ [BRAND_GUIDELINES](BRAND_GUIDELINES.md) 参照）。

---

## 6. アイキャッチ画像

- **比率 16:9・1600×900px 前後**を推奨。ビルド時にWebPへ自動最適化される。
- 文字を焼き込む場合は、一覧カード（16:10で軽くトリミング）で見切れないよう**文字を中央寄りの安全域**に置く。文字なし画像＋サイト側タイトル表示が最も破綻しにくい。
- 元画像は `src/content/blog/images/` に置き、フロントマターへ**記事からの相対パス**で指定。
- 生成AIで作る場合の指示例・注意は過去のやり取り参照（ChatGPTは横長=3:2固定など）。

---

## 7. 公開前チェックリスト

- [ ] `title` は分かりやすく、長すぎない
- [ ] `description` は100字前後で内容が伝わる
- [ ] `category` は6つのいずれか
- [ ] `tags` を付けた（任意）
- [ ] アイキャッチと `imageAlt` を設定（画像がある場合）
- [ ] 見出し（h2/h3）で構造化されている
- [ ] レビューなら「良い点/気になる点/向いている人」に触れた
- [ ] `npm run build` が通る
- [ ] `git push`（master へ）→ Cloudflare Pages が自動公開
